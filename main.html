1. 회원가입 및 로그인 데이터 흐름

1
회원가입 (Registration) 비밀번호를 Poseidon Hash로 변환하여 DB에 저장. 실제 비밀번호는 전송되지 않음.
전송 데이터: { ID, Poseidon_Hash }
2
로그인 요청 (Login Request) ID를 입력하면 DB에서 SavedHash를 브라우저로 가져옴.
3
증명 생성 (Client-side Proving) 브라우저 내에서 Password와 SavedHash를 사용하여 수학적 증명(Proof) 생성.
4
검증 요청 (Verification) 생성된 Proof만 Render 서버로 전송. 서버는 비밀번호를 보지 않고 참/거짓만 판독.

2. 영지식 증명(zk-SNARK) 구성 데이터

구분 구성 요소 (Data Assets) 상세 설명
Private Input password 브라우저 외부로 절대 유출되지 않는 사용자의 비밀값
Public Signal savedHash 검증의 기준이 되는 공개된 해시값 (인증 기준점)
Output Proof (pi_a, pi_b, pi_c) 비밀번호 정보를 포함하지 않은 수학적 계산 결과물
Verifier verification_key.json 서버가 증명의 유효성을 판단하는 고정된 수학적 잣대
